## Stack

Next.js 16 (App Router) + next-sanity 12, backed by Sanity project `qwwmf79r`
(dataset `production`). Hosted on Netlify (OpenNext adapter, auto-applied).
Migrated in-place from Gatsby on the `feat/migrate-to-nextjs` branch.

## Landmines

- Read `node_modules/next/dist/docs/` for Next 16 APIs — training data is stale
  (see `@AGENTS.md`). Next 16 request APIs are async and must be awaited.
- `sanity.types.ts` and `sanity.schema.json` are generated (typegen). Never edit
  manually — regenerated by `pnpm run sanity:typegen` (`predev`/`prebuild`).
- `sanity:typegen` runs `pnpm --dir ../bluepath-sanity exec sanity schema extract`
  — it requires the `bluepath-sanity` Studio repo checked out as a sibling
  directory. CI must replicate this layout.
- Build is Turbopack (Next 16 default). Do not add a custom webpack config or
  ship with `--webpack`.
- The `gatsby-*.ts(x)` files and `src/gatsby-types.d.ts` are dead code retained
  for rollback comparison; excluded from the TS build. Deleted in a post-cutover
  follow-up (spec §1.13). Do not import from them.
- Every styled-components consumer must be a Client Component (`'use client'`).
